## 4. 失败原因诊断与修复建议

### 🔍 失败根本原因

1. **步数限制过严**：
   - 当前最大步数为 20，而完整答题流程至少需要 20+ 步（登录 3 步 + 选课 2 步 + 进入错题集 2 步 + 20 题 × 1 步 + 提交 1 步 = 至少 28 步）。
   - 导致 Agent 在完成 9 题后被迫终止，无法继续。

2. **前期冗余操作消耗步数**：
   - 步骤 3 中尝试点击“年份筛选”属无效操作，浪费 1 步。
   - 若优化路径，可节省步数用于后续答题。

3. **缺乏批量答题机制**：
   - Agent 逐题点击，效率低。若支持“自动答题”或“批量提交”，可减少步骤消耗。

---

### 🛠 修复建议

| 问题类型 | 修复方案 |
|----------|----------|
| **步数限制** | 将最大步数提升至 35~40 步，确保覆盖完整答题流程。 |
| **操作路径优化** | 引导 Agent 优先识别显式按钮（如“点击进入”），避免无效筛选操作。 |
| **答题效率提升** | 支持“自动填充答案”或“一键提交”功能，减少单题操作次数。 |
| **容错机制增强** | 若某步失败（如元素未找到），应重试或跳过，而非阻塞整个流程。 |
