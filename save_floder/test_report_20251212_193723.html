4. 失败原因诊断与修复建议

🚫 风险点 1：元素索引依赖性强，易受 UI 变更影响

  • 问题描述：
  • Agent 多次依赖“元素索引”（如 index=20, 13, 40, 36）进行点击。
  • 当页面结构变更（如新增广告位、按钮位置调整），索引会失效，导致操作错误。
  • 修复建议：
  • 改用语义化定位（如 text="错题集" 或 aria-label="提交按钮"）。
  • 引入容错机制：若索引点击失败，自动尝试文本匹配或坐标点击。
  • 增加元素存在性校验：点击前先检查元素是否可见、可点击。

🚫 风险点 2：Step 6 误操作重复点击课程入口

  • 问题描述：
  • Agent 在进入课程后，未识别当前页面上下文，重复点击“点击进入”按钮。
  • 虽未导致失败，但属于冗余操作，增加执行时间与不确定性。
  • 修复建议：
  • 引入页面状态记忆机制：记录当前所在页面（如 /course/xxx），避免重复操作。
  • 使用页面标题或 URL 片段作为状态判断依据（如检测 title == "课程选择"）。

🚫 风险点 3：提示文案不完全一致

  • 问题描述：
  • 实际提示为：“请至少完成一道题目后再提交”
  • 预期提示为：“请完成所有题目后再提交！”

  • 若为产品需求，应统一提示文案。
  • 若为测试脚本预期，应在断言中放宽匹配规则（如模糊匹配“请完成...提交”）。
